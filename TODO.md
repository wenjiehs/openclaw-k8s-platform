## 阶段 M0：Operator 验证（第 1-2 周）

> **目标**：验证 OpenClaw Operator 在 TKE 环境的稳定性，决定是否采用 V2.0 方案
>
> **决策门槛**：
> - 部署成功率 > 95% → 继续 V2.0 Operator 方案
> - 部署成功率 < 90% → 降级到 V1.0 手动方案

### M0.1 Operator 部署（P0）

- [ ] **任务 M0.1.1：准备 TKE 测试集群**
  - 描述：创建独立的 TKE 测试集群，配置集群权限（RBAC），不影响生产环境
  - 验收标准：kubectl 可连接集群，节点状态 Ready
  - 预计工时：4 小时
  - 责任人：DevOps 工程师
  - 依赖：无

- [ ] **任务 M0.1.2：安装 cert-manager**
  - 描述：安装 cert-manager（v1.13.0+），用于自动申请 HTTPS 证书，验证 cert-manager Pod 正常运行
  - 验收标准：cert-manager Pod 状态 Running，可创建 ClusterIssuer
  - 预计工时：2 小时
  - 责任人：DevOps 工程师
  - 依赖：M0.1.1

- [ ] **任务 M0.1.3：安装 OpenClaw Operator**
  - 描述：通过 Helm 安装 OpenClaw Operator，配置 TKE 环境参数（cloudProvider=tencent），验证 Operator Pod 正常运行
  - 验收标准：Operator Pod 状态 Running，CRD 已注册（kubectl get crd | grep openclaw）
  - 预计工时：4 小时
  - 责任人：DevOps 工程师 + 后端工程师
  - 依赖：M0.1.1、M0.1.2

- [ ] **任务 M0.1.4：编写后端 K8s 适配代码**
  - 描述：实现 K8s Client（连接 TKE API），实现 OpenClawInstance CR 创建/查询/删除逻辑，编写单元测试
  - 验收标准：单元测试通过，可通过代码创建 CR 并查询 Status
  - 预计工时：16 小时
  - 责任人：后端工程师
  - 依赖：M0.1.3

**M0.1 小计**：26 小时（约 3.5 天）
